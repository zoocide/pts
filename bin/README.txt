Установка:
1. Скопировать файл test-dvm.conf в %userprofile%\.pts\test-dvm.conf
2. Заменить все пути в test-dvm.conf на свои.

Использвание:
1. В файл all.commands записать комманду, которая будет выполнена для всего набора тестов.
   Строки начинающиеся с # считаются комментарием.
   Комманды могут быть:
     compile ...
     run ...
   Например, комманде `dvm f -noH <имя теста>` соответствует `compile f -noH`.
   Указывать имя программы не нужно, так как оно будет приписано в конец для каждого теста свое.
   Комманде `dvm run 2 2 <имя теста>` соответствует `run 2 2`.
2. Запустить `dvm-test.pl -a`. В данный момент это единственный работающий вариант.
   В результате выполнения будет созданы описания задач в директории tasks.
   Если директория tasks отсутствует, ее необходимо создать до запуска скрипта, иначе будет выдана ошибка.
   Так же после успешного выполнения скрипта будет создан файл current.set, содержащий список задач, которые будут выполнены далее.
   Из файла current.set можно убрать или заккоментировать ненужные задачи.
3. Запустить `pts.pl -T tasks current.set`. Этот скрипт выполнит все задачи, записанные в current.set.
   pts.pl --help позволит посмотреть доступные опции для данного скрипта.
